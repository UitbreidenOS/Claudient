# 📂 Espacio de Trabajo de DevRel
> El espacio de trabajo canónico para un Defensor de Desarrolladores o Equipo de DevRel, diseñado para escalar el compromiso comunitario, automatizar la distribución de contenido técnico y rastrear sistemáticamente la fricción en la Experiencia del Desarrollador (DX).

📄 `devrel-charter.md`           # Resumen canónico: Personas de desarrollador objetivo, valores comunitarios centrales y OKRs principales (por ejemplo, contribuyentes activos versus conciencia de primera línea)
🧠 `active-campaigns-memory.md`  # Memoria de sesión: Seguimiento dinámico de contexto para hackathones actuales, CFPs activas y retroalimentación de lanzamiento beta
🤖 `CLAUDE.md`                   # Reglas operativas: Instrucciones estrictas para mantener un tono auténtico y centrado en desarrolladores (prohibir estrictamente el relleno de marketing, priorizar fragmentos de código y precisión técnica)

## 📁 content-and-tutorials/ (4 habilidades - Educación)
📄 `technical-blog-drafter.md`   # Marco para escribir tutoriales profundos centrados en arquitectura, casos límite y "cómo funciona bajo el capó"
📄 `cross-platform-publisher.md` # Reglas de formato automatizadas para sindicar un archivo markdown único de forma nativa a Dev.to, Hashnode, Medium y blogs personales
📄 `video-script-engine.md`      # Listas de tomas y esquemas de grabación de pantalla para demostraciones técnicas en YouTube de 5 minutos y clips cortos
📄 `demo-repo-scaffolder.md`     # Scripts para crear rápidamente repositorios GitHub boilerplate "Hello World" para codificación en vivo en conferencias

## 📁 community-and-events/ (4 habilidades - Compromiso)
📄 `cfp-abstract-generator.md`   # Redacta presentaciones de Call for Papers (CFP) convincentes adaptadas a temas específicos de conferencias tecnológicas
📄 `event-logistics-tracker.md`  # Manual para organizar encuentros, gestionar inventario de promocionales y coordinar viajes de oradores
📄 `ambassador-program.md`       # Pautas para identificar, recompensar y retener a los principales campeones de la comunidad y colaboradores de código abierto
📄 `discord-moderation-bot.md`   # Lógica para incorporación automatizada de servidores, filtrado de spam y enrutamiento de preguntas técnicas a los canales correctos

## 📁 developer-experience-dx/ (3 habilidades - Retroalimentación de Producto)
📄 `api-friction-logger.md`      # Plantilla sistemática para documentar los momentos exactos en que los desarrolladores se atascan durante la incorporación del SDK
📄 `feedback-triage-router.md`   # Convierte quejas comunitarias en Slack/Discord en tickets Jira formateados y procesables para el equipo de ingeniería
📄 `docs-teardown-auditor.md`    # Lista de verificación de revisión periódica para garantizar que las guías de "Primeros pasos" realmente funcionen con la última versión principal

## 📁 open-source-ops/ (3 habilidades - Ecosistema)
📄 `good-first-issue-tagger.md`  # Criterios para identificar y definir problemas abordables de GitHub para convertir observadores pasivos en colaboradores
📄 `release-notes-translator.md` # Transforma historiales de confirmación Git secos en anuncios de lanzamiento emocionantes y centrados en la comunidad
📄 `contributor-shoutouts.md`    # Agregación automatizada de PRs fusionados que no son de empleados para el boletín semanal

## 📁 metrics-and-impact/ (3 habilidades - Seguimiento del ROI)
📄 `utm-tag-generator.md`        # Estandariza enlaces de seguimiento en todas las publicaciones de blog, videos y códigos QR para atribución de canalización posterior
📄 `github-velocity-tracker.md`  # Monitorea el crecimiento de estrellas, proporciones de bifurcación y tiempos de resolución de problemas para evaluar la salud del proyecto
📄 `github-final-sync.md`        # Acción de CI/CD para confirmar blogs técnicos finalizados, cronogramas de eventos y código de demostración en repos finales de Github
