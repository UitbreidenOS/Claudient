# 📂 Workspace de DevRel
> El workspace canónico para un Developer Advocate o equipo de DevRel, diseñado para escalar la participación comunitaria, automatizar la distribución de contenido técnico y rastrear sistemáticamente la fricción en Developer Experience (DX).

📄 `devrel-charter.md`      # Resumen canónico: persona de desarrollador ICP, pilares de defensa y KPIs comunitarios
🧠 `active-initiatives.md`  # Memoria de sesión: contexto dinámico para campañas actuales de DevRel y calendario de eventos
🤖 `CLAUDE.md`              # Reglas operativas: instrucciones estrictas para escribir para desarrolladores senior y nunca sobreexplicar conceptos básicos

## 📁 community-engagement/ (4 skills - construir y nutrir comunidades de desarrolladores)
📄 `discord-community-ops.md`           # Automatizar la moderación de Discord, respuestas de FAQ y enrutamiento de preguntas de desarrolladores a expertos
📄 `github-discussions-strategy.md`     # Monitorear GitHub Discussions, etiquetar mantenedores relevantes y destacar puntos de fricción comunes en integración
📄 `community-event-planning.md`        # Orquestar meetups, talleres y hackathons con logística templada y análisis post-evento
📄 `ambassador-program.md`              # Reclutar, incorporar y rastrear embajadores comunitarios con estructuras de incentivos y métricas de impacto

## 📁 technical-content/ (4 skills - producir y distribuir contenido enfocado en desarrolladores)
📄 `content-calendar-management.md`     # Planificar, coordinar y publicar posts de blog, tutoriales y casos de estudio en múltiples canales
📄 `code-example-curation.md`           # Extraer, probar y documentar patrones de código de proyectos internos y contribuciones comunitarias
📄 `api-documentation-review.md`        # Auditar documentación de API para claridad, completitud y usabilidad para desarrolladores; rastrear brechas de DX
📄 `developer-newsletter.md`            # Curar boletines técnicos semanales/mensuales destacando actualizaciones de productos, logros comunitarios y recursos de aprendizaje

## 📁 developer-experience/ (3 skills - identificar y resolver fricción)
📄 `friction-tracking-system.md`        # Recopilar, categorizar y priorizar feedback de desarrolladores desde tickets de soporte, encuestas y canales comunitarios
📄 `onboarding-audit.md`                # Probar y documentar la experiencia completa de registro de desarrollador, autenticación y primera llamada a API
📄 `integration-troubleshooting.md`     # Diagnosticar fallos comunes de integración, documentar soluciones alternativas y retroalimentar aprendizajes a equipos de producto

## 📁 event-strategy/ (4 skills - planificar y amplificar eventos para desarrolladores)
📄 `conference-sponsorship-roi.md`      # Rastrear asistencia a conferencias, gestionar presencia de stand y medir impacto en pipeline
📄 `webinar-production.md`              # Programar oradores, gestionar configuración técnica, manejar moderación de Q&A y archivar contenido
📄 `office-hours-scheduling.md`         # Coordinar horas de oficina recurrentes para desarrolladores con oradores expertos rotativos y distribución de grabaciones
📄 `developer-relations-reporting.md`   # Agregar métricas comunitarias, resumir impacto mensual de DevRel e identificar oportunidades de crecimiento

## 📁 analytics-and-feedback/ (3 skills - medir e iterar)
📄 `community-health-dashboard.md`      # Rastrear estrellas de GitHub, crecimiento de Discord, asistencia a eventos e integración de contenido en múltiples plataformas
📄 `developer-sentiment-analysis.md`    # Analizar posts de foros, menciones de Twitter/X y comentarios de issues para identificar cambios en el sentimiento
📄 `feedback-synthesis-reports.md`      # Sintetizar feedback trimestral de todas las fuentes en recomendaciones accionables de producto y defensa

---
**Archivos de Configuración**
⚙️ `devrel-config.json`     # Configuración del workspace: URLs de dashboard, endpoints de webhook de Slack, integraciones de calendario de eventos
📦 `content-vault.md`       # Índice centralizado de todo el contenido publicado, muestras de código y enlaces de recursos con seguimiento de versiones
