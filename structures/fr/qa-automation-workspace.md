# 📂 Espace de Travail Devrel
> L'espace de travail canonique pour un Défenseur des Développeurs ou une Équipe DevRel, conçu pour adapter l'engagement communautaire, automatiser la distribution de contenu technique et suivre systématiquement les frictions de l'Expérience Développeur (DX).

📄 `devrel-charter.md`           # Brief canonique : Personas de développeurs cibles, valeurs communautaires fondamentales et OKRs primaires (par exemple, contributeurs actifs vs. notoriété en haut d'entonnoir)
🧠 `active-campaigns-memory.md`  # Mémoire de session : Suivi de contexte dynamique pour les hackathons actuels, les appels à papiers actifs et les retours de lancement bêta
🤖 `CLAUDE.md`                   # Règles d'exploitation : Instructions strictes pour maintenir un ton authentique et centré sur les développeurs (interdire strictement le contenu marketing, privilégier les extraits de code et la précision technique)

## 📁 content-and-tutorials/ (4 compétences - Éducation)
📄 `technical-blog-drafter.md`   # Cadre pour rédiger des tutoriels approfondis en mettant l'accent sur l'architecture, les cas limites et « comment ça fonctionne sous le capot »
📄 `cross-platform-publisher.md` # Règles de formatage automatisées pour syndiquer un seul fichier markdown de manière native vers Dev.to, Hashnode, Medium et des blogs personnels
📄 `video-script-engine.md`      # Listes de plans et contours d'enregistrement d'écran pour des démos YouTube techniques de 5 minutes et des courts-métrages
📄 `demo-repo-scaffolder.md`     # Scripts pour initialiser rapidement des dépôts GitHub « Hello World » pour les démonstrations de codage en direct lors de conférences

## 📁 community-and-events/ (4 compétences - Engagement)
📄 `cfp-abstract-generator.md`   # Rédige des propositions d'appels à papiers convaincantes adaptées aux thèmes spécifiques des conférences technologiques
📄 `event-logistics-tracker.md`  # Playbook pour organiser des rencontres, gérer l'inventaire des cadeaux et coordonner les déplacements des orateurs
📄 `ambassador-program.md`       # Directives pour identifier, récompenser et fidéliser les meilleurs champions communautaires et contributeurs open-source
📄 `discord-moderation-bot.md`   # Logique pour l'intégration automatisée du serveur, le filtrage du spam et l'acheminement des questions techniques vers les bons canaux

## 📁 developer-experience-dx/ (3 compétences - Retour d'Information Produit)
📄 `api-friction-logger.md`      # Modèle systématique pour documenter les moments exacts où les développeurs se bloquent lors de l'intégration du SDK
📄 `feedback-triage-router.md`   # Convertit les plaintes communautaires dans Slack/Discord en tickets Jira formatés et exploitables pour l'équipe d'ingénierie
📄 `docs-teardown-auditor.md`    # Liste de vérification d'examen périodique pour assurer que les guides 'Démarrage Rapide' fonctionnent réellement avec la dernière version majeure

## 📁 open-source-ops/ (3 compétences - Écosystème)
📄 `good-first-issue-tagger.md`  # Critères pour identifier et délimiter les problèmes GitHub accessibles afin de convertir les lurkers en contributeurs
📄 `release-notes-translator.md` # Transforme les historiques Git secs en annonces de version excitantes et centrées sur la communauté
📄 `contributor-shoutouts.md`    # Agrégation automatisée des PRs fusionnées non-employés pour la newsletter hebdomadaire

## 📁 metrics-and-impact/ (3 compétences - Suivi du Retour sur Investissement)
📄 `utm-tag-generator.md`        # Standardise les liens de suivi dans tous les articles de blog, vidéos et codes QR pour l'attribution du pipeline en aval
📄 `github-velocity-tracker.md`  # Surveille la croissance des étoiles, les ratios de fork et les temps de résolution des problèmes pour évaluer la santé du projet
📄 `github-final-sync.md`        # Action CI/CD pour valider les blogs techniques, les calendriers d'événements et le code de démonstration dans les dépôts finaux de Github
