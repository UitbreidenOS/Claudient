# 📂 Espace de travail DevRel
> L'espace de travail canonique pour un Developer Advocate ou une équipe DevRel, conçu pour mettre à l'échelle l'engagement communautaire, automatiser la distribution de contenu technique et suivre systématiquement les frictions de l'expérience développeur (DX).

📄 `devrel-charter.md`      # Document de référence : persona ICP du développeur, piliers de défense et KPI communautaires
🧠 `active-initiatives.md`  # Mémoire de session : contexte dynamique pour les campagnes DevRel actuelles et calendrier d'événements
🤖 `CLAUDE.md`              # Règles d'exploitation : instructions strictes pour écrire à l'intention des développeurs seniors et ne jamais sur-expliquer les concepts basiques

## 📁 community-engagement/ (4 compétences - construire et nourrir les communautés de développeurs)
📄 `discord-community-ops.md`           # Automatiser la modération Discord, les réponses FAQ et l'acheminement des questions des développeurs vers les experts
📄 `github-discussions-strategy.md`     # Surveiller les discussions GitHub, identifier les mainteneurs pertinents et mettre en évidence les points de douleur d'intégration courants
📄 `community-event-planning.md`        # Orchestrer les rencontres, ateliers et hackathons avec une logistique modélisée et une analyse post-événement
📄 `ambassador-program.md`              # Recruter, intégrer et suivre les ambassadeurs communautaires avec des structures d'incitations et des métriques d'impact

## 📁 technical-content/ (4 compétences - produire et distribuer du contenu axé sur les développeurs)
📄 `content-calendar-management.md`     # Planifier, coordonner et publier des articles de blog, tutoriels et études de cas sur tous les canaux
📄 `code-example-curation.md`           # Extraire, tester et documenter les modèles de code provenant des projets internes et des contributions communautaires
📄 `api-documentation-review.md`        # Auditer les docs API pour la clarté, l'exhaustivité et l'utilisabilité pour les développeurs ; suivre les lacunes DX
📄 `developer-newsletter.md`            # Organiser des newsletters techniques hebdomadaires ou mensuelles mettant en évidence les mises à jour de produits, les réussites communautaires et les ressources d'apprentissage

## 📁 developer-experience/ (3 compétences - identifier et résoudre les frictions)
📄 `friction-tracking-system.md`        # Collecter, catégoriser et prioriser les commentaires des développeurs provenant des tickets d'assistance, des sondages et des canaux communautaires
📄 `onboarding-audit.md`                # Tester et documenter l'expérience complète d'inscription, d'authentification et du premier appel API du développeur
📄 `integration-troubleshooting.md`     # Diagnostiquer les défaillances d'intégration courantes, documenter les solutions de contournement et transmettre les apprentissages aux équipes produit

## 📁 event-strategy/ (4 compétences - planifier et amplifier les événements pour développeurs)
📄 `conference-sponsorship-roi.md`      # Suivre l'assistance aux conférences, gérer la présence du stand et mesurer l'impact sur le pipeline
📄 `webinar-production.md`              # Planifier les conférenciers, gérer la configuration technique, gérer la modération des Q&A et archiver le contenu
📄 `office-hours-scheduling.md`         # Coordonner les heures de bureau récurrentes pour les développeurs avec des conférenciers experts rotatifs et la distribution des enregistrements
📄 `developer-relations-reporting.md`   # Agréger les métriques communautaires, résumer l'impact mensuel de DevRel et identifier les opportunités de croissance

## 📁 analytics-and-feedback/ (3 compétences - mesurer et itérer)
📄 `community-health-dashboard.md`      # Suivre les stars GitHub, la croissance Discord, l'assistance aux événements et l'engagement du contenu sur tous les plateformes
📄 `developer-sentiment-analysis.md`    # Analyser les publications sur les forums, les mentions Twitter/X et les commentaires d'issues pour identifier les changements de sentiment
📄 `feedback-synthesis-reports.md`      # Synthétiser les retours trimestriels de toutes les sources en recommandations produit et d'défense exploitables

---
**Fichiers de configuration**
⚙️ `devrel-config.json`     # Paramètres d'espace de travail : URL de tableau de bord, points de terminaison webhook Slack, intégrations de calendrier d'événements
📦 `content-vault.md`       # Index centralisé de tout le contenu publié, les exemples de code et les liens de ressources avec suivi de version
