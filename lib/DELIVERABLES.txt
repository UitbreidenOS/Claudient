File: lib/goal-parser.js
Size: 11 KB (389 lines)
Type: Production-ready JavaScript module

Purpose:
  Parse natural-language engineering goals into structured project plans with
  subtasks, dependencies, and acceptance criteria.

Features:
  ✓ 5 Regex patterns (add, distribute, integrate, build, setup)
  ✓ 7 Domain keyword categories (auth, infrastructure, database, api, testing, docs, deployment)
  ✓ Automatic quantifier extraction (e.g., "15 microservices")
  ✓ Dependency inference based on goal content
  ✓ Phase-based subtask generation (planning → implementation → rollout → testing → deployment)
  ✓ Context-aware acceptance criteria
  ✓ Complexity scoring (low/medium/high)
  ✓ Scope identification

API:
  parseGoal(goalText: string): ParsedGoal
  parseGoals(goals: string[]): ParsedGoal[]
  formatGoal(parsed: ParsedGoal): string

Example Input:
  "Add OAuth2 + SAML across 15 microservices"

Example Output:
  {
    goal: "Add OAuth2 + SAML across 15 microservices",
    pattern: "add",
    components: {
      quantifier: { count: 15, unit: "microservices" },
      categories: { auth: [...], infrastructure: [...] }
    },
    subtasks: [5 tasks with dependencies],
    dependencies: [5 inferred dependencies],
    acceptance_criteria: [13 criteria],
    metadata: {
      complexity: "medium",
      scope: "15 microservices",
      estimated_phases: ["planning", "implementation", "rollout", "testing", "deployment"]
    }
  }

================================================================================
PATTERNS RECOGNIZED
================================================================================

1. ADD PATTERN
   Regex: /^(?:add|implement|integrate|enable)\s+([^,]+?)\s+(?:to|across|on|in)\s+(.+)$/i
   Examples:
     "Add OAuth2 to API"
     "Add OAuth2 + SAML across 15 microservices"
     "Implement rate limiting across all services"

2. DISTRIBUTE PATTERN
   Regex: /^(?:implement|deploy|integrate|add)\s+([^,]+?)\s+across\s+(\d+)\s+([^.]+?)(?:\.|$)/i
   Examples:
     "Implement feature flags across 20 services"
     "Deploy monitoring across 8 clusters"

3. INTEGRATE PATTERN
   Regex: /^(?:integrate|combine|merge|add)\s+([^.]+?)\s+(?:and|with)\s+([^.]+?)(?:\s+(?:to|across|in|on)\s+(.+))?$/i
   Examples:
     "Integrate Redis and PostgreSQL"
     "Integrate A and B across all services"

4. BUILD PATTERN
   Regex: /^(?:build|create|develop|implement)\s+([^,]+?)\s+for\s+(.+)$/i
   Examples:
     "Build authentication layer for mobile app"

5. SETUP PATTERN
   Regex: /^(?:setup|configure|install|enable)\s+([^,]+?)\s+with\s+(.+)$/i
   Examples:
     "Setup PostgreSQL with replication"

================================================================================
EXECUTION PHASES
================================================================================

1. PLANNING (20% effort)
   Task: Architecture & Design Review
   Description: Document architecture, security requirements, and integration points

2. IMPLEMENTATION (40% effort)
   Tasks vary by category:
   - Auth: Implement Authentication/Authorization
   - Database: Database Schema Updates
   - API: Update API Endpoints

3. ROLLOUT (10% effort, multi-service goals only)
   Task: Rollout across N services
   Description: Systematically apply changes to all instances

4. TESTING (20% effort)
   Task: Security & Integration Testing
   Description: Unit, integration, and security testing across all affected components

5. DOCUMENTATION (5% effort)
   Task: Update Documentation
   Description: Document changes, deployment procedures, and usage guides

6. DEPLOYMENT (5% effort)
   Task: Staged Deployment & Monitoring
   Description: Deploy to staging, validate, then production rollout

================================================================================
ACCEPTANCE CRITERIA LOGIC
================================================================================

Universal criteria (all goals):
  ✓ All code changes peer-reviewed and approved
  ✓ Automated tests pass (unit + integration)
  ✓ Runbooks and troubleshooting guides documented
  ✓ Team trained on new features/changes
  ✓ Monitoring and alerting configured
  ✓ Metrics baseline established

Auth-specific:
  ✓ Authentication mechanism verified working
  ✓ Authorization policies enforced across all endpoints
  ✓ Security audit completed and signed off
  ✓ No unauthorized access vulnerabilities found

Database-specific:
  ✓ Database migrations tested and reversible
  ✓ Data integrity verified post-migration

API-specific:
  ✓ All API endpoints tested and documented
  ✓ Client compatibility confirmed

Multi-service distribution:
  ✓ Deployed successfully to all N services
  ✓ Zero-downtime deployment verified
  ✓ All instances in healthy state post-deployment

================================================================================
COMPLEXITY SCORING
================================================================================

LOW (1-3 subtasks):
  Simple feature additions with minimal dependencies
  Examples: "Add rate limiting", "Implement feature flags"
  Typical timeline: 1-2 weeks

MEDIUM (4-5 subtasks):
  Multi-component changes affecting multiple services
  Examples: "Add OAuth2 to 15 microservices"
  Typical timeline: 2-4 weeks

HIGH (6+ subtasks):
  Enterprise-scale initiatives with complex coordination
  Examples: "Migrate database across all regions"
  Typical timeline: 4+ weeks

USAGE EXAMPLES

Example 1: Basic goal parsing

  const result = parseGoal('Add OAuth2 + SAML across 15 microservices');
  console.log(result.metadata.complexity); // "medium"

Example 2: Batch processing
  const { parseGoals } = require('./lib/goal-parser');
  const results = parseGoals(['Add OAuth2 to API', 'Deploy Redis']);

Example 3: Format for display
  const { parseGoal, formatGoal } = require('./lib/goal-parser');
  console.log(formatGoal(parseGoal('Add OAuth2 across 15 services')));

Example 4: Extract subtasks by phase
  const result = parseGoal('...');
  const implTasks = result.subtasks.filter(t => t.phase === 'implementation');

Example 5: Get acceptance criteria
  const result = parseGoal('...');
  result.acceptance_criteria.forEach((criterion, idx) => {
    console.log(`[ ] ${idx + 1}. ${criterion}`);
  });
