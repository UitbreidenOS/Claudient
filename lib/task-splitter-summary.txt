CORE FEATURES IMPLEMENTED:
==========================

1. TASK COMPLEXITY ANALYSIS
   ✓ Keyword-based complexity detection
   ✓ Time estimation based on:
     - Description keywords (network, database, file I/O, transactions, async, etc.)
     - Number of sub-tasks
     - Number of files
     - Concurrency indicators
   ✓ Returns: complexity level + estimated minutes + detailed reason

2. SMART SPLIT SUGGESTIONS
   ✓ Automatic detection when task > 30 minutes
   ✓ Multiple split strategies:
     - By items (extract from description)
     - By batch (group into smaller chunks)
     - By sequence (create ordered stages)
     - By type (group by category)
   ✓ Returns rationale and prioritized suggestions

3. AUTO-SPLIT & PARALLEL EXECUTION
   ✓ Automatic strategy inference
   ✓ Configurable concurrency (default: 4)
   ✓ Task timeout support (default: 30 min)
   ✓ Detailed execution tracking
   ✓ Returns: completed count, failed count, results map, failures

4. AUTOMATIC RETRY ON FAILURE
   ✓ Initial attempt as-is
   ✓ On failure: auto-split and retry components
   ✓ Retry failed components individually
   ✓ Configurable max retries (default: 3)
   ✓ Returns: success status, results, failures, retry count

5. DEPENDENCY ANALYSIS
   ✓ Build dependency graph
   ✓ Identify independent vs dependent tasks
   ✓ Supports multiple dependency formats:
     - dependencies: string[]
     - dependsOn: string[]
     - order: number
   ✓ Returns: independent tasks, dependent tasks, full graph


PUBLIC API:
===========

Constructor:
  new TaskSplitter(options)
    - timeoutMs: number (default: 30 * 60 * 1000)
    - maxRetries: number (default: 3)
    - concurrency: number (default: 4)
    - verbose: boolean (default: false)

Methods:
  analyzeTask(task) -> AnalysisResult
  suggestSplits(task) -> SplitSuggestion
  autoSplit(task) -> SplitResult
  executeParallel(subtasks, executor) -> ExecutionResult
  executeWithRetry(task, executor) -> RetryResult
  analyzeDependencies(tasks) -> DependencyAnalysis

Events:
  .on('log', msg => ...)


TASK SCHEMA:
============

{
  id: string (required)
  name: string (required)
  description: string (optional)
  parent: string (optional - for hierarchical tasks)
  type: string (optional - for grouping)
  order: number (optional - for sequencing)
  subtasks: Task[] (optional - child tasks)
  dependencies: string[] (optional - task IDs)
  dependsOn: string[] (optional - alternative)
  files: string[] (optional - file paths)
  status: 'pending' | 'completed' | 'failed' (optional)
}


HEURISTICS & ALGORITHMS:
=========================

Task Complexity Scoring:
- API/network keywords: +10 min
- Database operations: +8 min
- File I/O: +5 min
- Data transformation: +3 min
- Validation/testing: +5 min
- Error handling: +4 min
- Large scope (>500 chars): +10 min
- Per sub-task: +2 min
- Per 10+ files: +5 min
- Default minimum: 1 min

Split Strategies (in order of inference):
1. Existing subtasks: split into sub-tasks array
2. Multiple/several/each keywords: split by items
3. Sequential keywords (first/then/next): split by sequence
4. Type/category keywords: split by type
5. Fallback: split by batch

Concurrency Model:
- Maintains queue of tasks
- Fills concurrency slots up to limit
- Waits for completion before adding more
- Continues until queue empty and all tasks complete

USAGE EXAMPLE:
==============

const { TaskSplitter } = require('./lib/task-splitter')

const splitter = new TaskSplitter({
  concurrency: 4,
  verbose: true
})

const task = {
  id: 'batch-import',
  name: 'Import Data Files',
  description: 'Process CSV files with concurrent validation and transformation',
  subtasks: [
    { id: 'csv-1', file: 'users.csv' },
    { id: 'csv-2', file: 'orders.csv' },
    { id: 'csv-3', file: 'products.csv' }
  ]
}

// Get insights
const analysis = splitter.analyzeTask(task)
console.log(`Task complexity: ${analysis.complexity}`)

// Auto-split
const split = splitter.autoSplit(task)
console.log(`Split into ${split.subtasks.length} parallel tasks`)

// Execute
const executor = async (t) => processFile(t.file)
const result = await splitter.executeParallel(split.subtasks, executor)
console.log(`Completed: ${result.completed}, Failed: ${result.failed}`)

PERFORMANCE:
=============

Time Complexity:
- Task analysis: O(description length + subtasks count)
- Auto-split: O(n) where n = number of subtasks
- Parallel execution: O(n/concurrency) wall-clock time
- Dependency analysis: O(n + edges) where n = tasks

Space Complexity:
- Dependency graph: O(n + edges)
- Execution results: O(n * result_size)
- Queue and processing: O(concurrency)
